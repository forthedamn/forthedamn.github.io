Dom学习之Dom类型介绍

在开发中我们会接触到很多dom操作，熟悉dom节点的类型是必须掌握的知识点。
首先，来看看看dom节点一共有哪些常见的类型：

节点类型 数值常量 字符常量
Element(元素节点) 1 ELEMENT_NODE
Attr(属性节点) 2 ATTRIBUTE_NODE
TEXT(文本节点) 3 TEXT_NODE
Comment(注释节点) 8 COMMENT_NODE
Document(文档节点) 9 DOCUMENT_NODE
DocumentType(文档类型节点) 10 DOCUMENT_TYPE_NODE
DocuemntFragment(文档片段节点) 11 DOCUMENT_FRAGMENT_NODE

表中的数值常量是怎么获得的？
通过调用节点的nodeType方法获得nodeType值：
var divNode = document.getElementsByTagName('div')[0]
divNode.nodeType

表中的字符常量是什么？
可以通过Node.ELEMENT_NODE获得。
其中，Node是一个在客户端Javascript实现的东西，你可以理解为是浏览器为了更好的处理dom，创建了这样一个对象，这个Node对象是所有节点对象的父类型。在低版本的ie浏览器中，并不存在这个Node对象，没法用字符常量来区分节点，所有一般比较节点采用数值常量。

Element(元素节点)

元素节点是指一个html元素，比如div,span,nav标签等
var divNode = document.getElementsByTagName('div')[0]
divNode.nodeTyp
比如这样获得的节点就是一个Element节点。

Attr(属性节点)

属性节点是指html元素的属性，比如 <div class='left'></div>,其中的class='left'就是一个属性节点。
我们可以这样获得一个属性节点(比如说class)
var divNode = document.getElementsByTagName('div')[0]
var classNode = divNode.attributes.class

TEXT(文本节点)

文本节点是指html文档中出现的文本，比如 <div class='left'>123</div>，其中的123就是一个文本节点，另外代码之间的空格，回车等也会形成一个文本节点。

    <div>text</div>
    var divNode = document.getElementsByTagName('div')[0];
    var divNode_childNodes = divNode.childNodes;

其中divNode_childNodes就包括了文本节点，如果这个divNode在写代码时有换行，那么换行的这个‘回车’也是一个textNode。
ps：在有些时候，多个元素布局之间存在缝隙或者其他问题，有可能就是由于这个文本节点的存在引起的。
比如:
    div{
        display: inline-block;
        width:50px;
        height: 50px;
    }
    <div></div>
    <div></div>
    <div></div>
    <div></div>

这几个div之间会有空隙，就算根本没有设置任何margin，padding。
这个时候的解决办法是取消div之间的文本节点。像这样
    <div></div><div></div><div></div><div></div>

Comment(注释节点)

注释节点，故名思意，就是html文档中的注释，注释不属于文本节点。
同样可以像获取文本节点那样获取注释节点。

Document(文档节点)

文档节点，这个节点指的是整个文档document。
ps：document就是整个dom树的根节点，也是Node节点的子节点哦

DocumentType(文档类型节点)

文档类型节点，比如<!DOCTYPE html>就是一个文档类型节点

DocuemntFragment(文档片段节点)

文档片段节点，这个节点没有父节点，一般用来缓存、临时操作一些dom。
DocuemntFragment是存在内存中的对象，使用DocuemntFragment代替一些dom操作可以提升页面的性能,比如：

    // 不使用DocuemntFragment
    for(let i = 0,ii=100;i<ii;i++) {
        var div = document.createElement('div');
        div.innerText = i;
        // 这里涉及到了大量的页面重绘和回流，降低性能
        document.body.appendChild(div);
    }
    // 使用DocuemntFragment
    var frag = document.createDocumentFragment();
    for(let i = 0,ii=100;i<ii;i++) {
        var div = document.createElement('div');
        div.innerText = i;
        // frag是内存中对象，没有涉及到页面渲染这一层，速度很快。
        frag.body.appendChild(div);
    }
    // 最后一次性加入当页面中
    document.body.appendChild(frag);

最后，再来介绍之前在文章中出现的一个知识点 NodeList。

之前在文章中看到了var divNode_childNodes = divNode.childNodes;，这个divNode_childNodes就是一个NodeList。
什么是NodeList？

NodeList 对象是一个节点的集合，是由 Node.childNodes 和 document.querySelectorAll 返回的.
NodeList.length 返回其中节点的数量
NodeList.item(i) 返回引索为i的节点，如果越界，则返回null
NodeList[i] 返回引索为i的节点，如果越界，则返回undefine

NodeList跟数组看起来很像，但NodeList不是数组。

NodeList和数组的区别

1.NodeList不是一个数组，数组的方法（map，forEach）在这里是不可用的，除非自己实现。
2.NodeList和数组在继承上的区别：
myArray --> Array.prototype --> Object.prototype --> null
myNodeList --> NodeList.prototype --> Object.prototype --> null
3.NodeList可以使用list[0],list[1],这种方法检索，超出范围时返回undefine
4.NodeList具有length属性，返回NodeList中节点的个数
5.NodeList有个特别方法item(),这个方法接受引索值，可以起到和‘2’中一样的效果，不同点是如果引索值超出范围，item返回null
6.NodeList如何遍历：
for(var i = 0,ii=list.length;i<ii;i++) {}
如果使用for in 进行遍历，将会获得length，item属性，起不到遍历list的作用
